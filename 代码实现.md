## 第一步：数据处理

### 优化内容和格式，目的是为了让语句连贯，轮次清晰，能够用来进行基础对话训练和测量

```
cd data_processing
cd 内容格式筛选
python 内容格式处理
```


内容格式筛选从上到下分为10层
目标数据集：华为原始对话数据集
方法：Rule-based

Level 1: 内容筛选：无效案例链接 --> 去除含有不在案例库里的案例的对话，对存在案例的对话把案例网址 替换成[案例链接: {case ID}]

去除准确率：100%
保留准确率：100%


Level 2: 内容筛选：非文字信息 --> 把含有非文字信息的图片，链接，edoc等无法拿到具体语义信息的对话都去掉

去除准确率：96% --> 错误原因：有一些客服回答让点击网站的，这种有些是固定网站而不是链接 
保留准确率：96& --> 错误原因：有一些带表情包的对话没有去掉，有些影响表达，有些不影响


Level 3: 无效话术或者系统自动回复删除

去除准确率：100%
保留准确率：100%


Level 4： 重复信息 --> 去除 开头的话说两边，或者打错字重写，以及中英文重复

去除准确率：99% --> 错误原因：刚好用户重复两句接了一个客服的中英文重复
保留准确率：100%


Level 5： 信息脱敏 --> 只针对于工号，电话和邮箱进行脱敏

去除准确率：无
保留准确率：89% --> 错误原因：直接给工号, 给名字，给其他人的名字和联系方式 这些都无法识别


Level 6：标点修正 --> 对没有标点符号的断句之家 补上标点符号 更加的规范化格式化

去除准确率：无
保留准确率：100%


Level 7：用户开头 --> 删除客服开头 直至用户开头 ，同时删除只有用户的对话

去除准确率：100%
保留准确率：100%


Level 8：客服结尾 --> 从后往前删除客服结尾 直至用户结尾 ，同时删除只有客服的对话

去除准确率：100%
保留准确率：100%


Level 9：轮次交互 --> 整理格式至 用户-客服 作为一轮对话的 轮次交互

去除准确率：无
保留准确率：100%


Level 10: 中文用户 --> 去掉英文的用户

去除准确率：94%  --> 错误原因：有些用户说数字+字母组合，比如：165G，或者打了 拼音，比如nihao
保留准确率：100%

### 案例推荐筛选

第一层：带有caseId的对话

第二层：case Id能够解决的对话（我们通过每轮依次标注）

首先，输入u1 ，过query模型输出query1，过RAG，得到evidence1，然后根据完整对话判断是否需要调e1（no-retrieval， re-retrieval，continue-to-evidence） 得到标签，并且更新当前对话evidence

输入u1， s1， e1，u2，过query模型输出query2，过RAG，得到e2，然后根据完整对话判断是否需要调e2（no-retrieval， re-retrieval，continue-to-evidence） 得到标签，并且更新当前对话evidence
